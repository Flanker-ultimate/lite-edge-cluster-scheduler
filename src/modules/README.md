主要用于演示和验证系统功能。它分为三个子模块：

## 任务请求端 (`src/modules/task_producer`)
这个模块模拟了客户端（Client），负责产生任务并接收结果。
* 功能：模拟外部用户向集群提交计算任务。
* 核心组件：
    * `send_pic.py` (图片发送器)：遍历本地文件夹中的图片，通过 HTTP 请求将其发送给 Master 节点的调度中心。这模拟了实际场景中监控摄像头或用户上传图片流的行为。
    * `rst_recv.py` (结果接收器)：用于接收处理完成后的结果（如推理后的坐标、类别或处理过的图片）。

## 管理端组件 (`src/modules/master`)
这个模块包含 Master 节点上辅助调度器工作的业务脚本。
* 功能：作为任务数据的“中转站”或“预处理器”。
* 核心组件：
    * `pic_manager.py` (图片管理器)：它接收来自 `task_producer` 的图片数据，并将其转发给 Master 核心调度器（Gateway/Scheduler）。它起到了将业务数据流与调度控制流对接的作用。

## 计算节点端 (`src/modules/slave`)
这个模块运行在边缘计算设备（Slave Agent）上，包含具体的计算任务实现和相关服务。
功能：执行具体的 AI 推理或图像处理任务，并返回结果。
核心组件：
* `task_dealer/可执行程序/yolov5-ascend/` (具体任务实现)：这是一个完整的 YOLOv5 推理工程，适配华为昇腾（Ascend 310/Atlas 300I）硬件。它包含模型转换指南、推理脚本 (`detect_yolov5_ascend.py`) 和相关依赖，用于在边缘端实际执行目标检测任务。
* `recv_server.py` (任务接收服务)：在计算节点上启动一个微型 HTTP 服务器，等待 Master 网关将调度过来的图片文件发送给它。
* `rst_send.py` (结果回传)：推理完成后，调用此脚本将计算结果发回给客户端或存储端。
